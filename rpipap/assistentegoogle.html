Vamos iniciar o Projeto

1.1 Antes de começarmos a configurar o código do Assistente da Google no próprio Raspberry Pi, primeiro devemos registar e configurar um projeto no Google Actions Console. Com a sua conta do Google pronta para ir para o painel de ações do Google Console , ou simplesmente clique aqui Google Actions Console.

Depois de fazer login com a sua conta, será redirecionado para a seguinte tela.
( 1. ) Aqui vai clicar no botão " Novo projeto ", como mostra a captura de tela abaixo. será solicitado a inserir um " Nome do projeto "
( 2. ) Além do nome do projeto, precisa definir seu país e seu idioma, conforme mostra a captura de ecrã
( 3. ) Depois de definir o nome do projeto e escolher o idioma e país, clique no botão " Criar projeto " (Create Project) .

( 2. ) Depois de ter certeza de que seu projeto atual está selecionado, clique no botão " Ativar "

Vamos começar a Programar




2. Configurar o áudio para o Google Assistant



2.1. Agora que configuramos uma conta no Google Actions Console, devemos configurar o áudio para O Raspberry. O SDK do Assistente do Google que usaremos possui alguns requisitos para que funcione corretamente. Para começar a configurar o áudio no Raspberry Pi, primeiro precisamos obter os números do cartão e do dispositivo para as várias entradas e saídas. As etapas abaixo mostram o que é preciso para obter os números corretos para esses dispositivos.


2.2. Localize o microfone USB utilizando o seguinte comando. Anote o número do cartão e o número do dispositivo. 


arecord -l

2.3. Agora, para localizar seu alto-falante, utilizaremos o seguinte comando. Anote novamente o número do cartão e o número do dispositivo. Observe que o conector de 3,5 mm do Raspberry Pi é rotulado como Analógico ou bcm2835 ALSA , com a saída HDMI identificada como bcm2835 IEC958 / HDMI . 


aplay -l

2.4. Agora que pegamos o número de dispositivo e cartão para o microfone e a saída de áudio, precisamos de criar um arquivo chamado ".asoundrc" no diretório inicial de usuários do pi. O arquivo .asoundrc ajuda a definir quais dispositivos de áudio o driver de áudio deve utilizar. Podemos criar esse arquivo executando o seguinte comando no terminal. 


nano /home/pi/.asoundrc

2.5. Dentro deste arquivo, insira as seguintes linhas. Certifique-se de substituir "número do cartão" e por no "número do dispositivo" os respectivos valores que recuperou durante a Etapa 2 . 


pcm.!default {
type asym
capture.pcm "mic"
playback.pcm "speaker"
}
pcm.mic {
type plug
slave {
pcm "hw:,"
}
}
pcm.speaker {
type plug
slave {
pcm "hw:,"
}
}

2.6. Com as linhas inseridas e as alterações feitas. Salve pressionando CTRL + X , Y e, finalmente, ENTER .



3. Download e configuração do Google Assistant - Teste de Som/Gravação



3.1. Antes de começarmos todo o trabalho duro de configurar o Raspberry Pi Google Assistant e de configurar a API necessária, testamos primeiro para garantir que o áudio esteja a funcionar. Nesta fase, você deve ter o microfone e os alto-falantes USB conectados ao seu Raspberry Pi. Depois de ter a certeza de que ambos estão conectados ao Raspberry Pi, podemos testar para garantir que os alto-falantes estejam a funcionar corretamente executando o seguinte comando.


speaker-test -t wav

Deve ouvir o som dos alto-falantes. Este som será uma pessoa falando. Se não ouvir nada da verificação dupla do alto-falante, eles serão conectados corretamente e ativados.

3.2. Agora, vamos testar nosso microfone a gravar. Para isso, executaremos o seguinte comando no seu Raspberry Pi. Este comando fará uma gravação curta de 5 segundos. 


arecord --format=S16_LE --duration=5 --rate=16000 --file-type=raw out.raw

Se receber um erro ao executar este comando, verifique se o microfone está conectado, mas este comando será bem-sucedido se conseguir ouvir com êxito o microfone.

4. Download e configuração do Google Assistant - Instalação



4.1. Com a API do Assistente do Google agora configurada, existem algumas coisas que precisamos fazer. Observe que, a partir desta parte, deverá concluir o tutorial na área de trabalho Raspbian diretamente e não sobre SSH, pois de precisar usar o navegador da web incorporado. Vamos primeiro atualizar a lista de pacotes do Raspberry Pi executando o seguinte comando.


sudo apt-get update

4.2. Após a atualização do Raspberry Pi, podemos prosseguir com a configuração de tudo o que precisamos para executar a API do Google Assistant. No Raspberry Pi, criaremos um arquivo no qual armazenaremos as credenciais que baixamos anteriormente no nosso computador. Para fazer isso, execute os dois comandos a seguir para criar uma pasta e começar a gravar nosso arquivo de credenciais. 


mkdir ~/googleassistant
nano ~/googleassistant/credentials.json

4.3. Nesse arquivo, precisamos copiar o conteúdo do arquivo de credenciais que baixamos para o computador. Pode abrir o arquivo .json em qualquer editor de texto e pressionar CTRL + A e CTRL + C para copiar o conteúdo. Agora, na sua janela SSH, clique com o botão direito do rato e clique em "Colar" .


4.4. Depois de copiar o conteúdo de suas credenciais para a nossa nano sessão, podemos salvar o arquivo pressionando Ctrl + X e Y e, finalmente, pressionando Enter .

4.5. Agora, com o arquivo de credenciais agora salvo com segurança no Raspberry Pi, iremos começar a instalar algumas das dependências nas quais confiamos. Execute o seguinte comando para instalar o Python3 e o ambiente virtual Python 3 no Raspberry Pi. 


sudo  apt-get install python3-dev python3-venv libssl-dev libffi-dev libportaudio2

4.6. Agora podemos ativar o python3 como a variável de ambiente virtual executando o seguinte comando no Raspberry Pi. 


python3   -m venv env

4.7. Com isso agora ativado, podemos prosseguir e garantir que instalamos as versões mais recentes do pip e do setuptools . Para fazer isso, executaremos o seguinte comando no Raspberry Pi. 


env/bin/python3   -m pip install --upgrade pip setuptools --upgrade

4.8. Para entrar neste novo ambiente Python que configuramos, devemos executar o seguinte comando no terminal.Para entrar neste novo ambiente Python que configuramos, devemos executar o seguinte comando no terminal. 


source    env/bin/activate

4.9. Agora que temos todos os pacotes necessários para instalar a Biblioteca do Assistente do Google, para isso, executaremos o seguinte comando para utilizar o pip para instalar a versão mais recente do pacote Python. 


python3  -m pip install --upgrade google-assistant-library
python3  -m pip install --upgrade google-assistant-sdk[samples]

5. Autorizando o Raspberry Pi para o Assistente do Google



5.1. Agora que configuramos todos os pré-requisitos para a execução do software Google Assistant no Raspberry Pi, podemos finalmente concluir as últimas etapas para autorizar o sistema. Para fazer isso, primeiro precisamos instalar a ferramenta de autorização do Google no nosso Raspberry Pi. Este pacote irá permitir-nos autenticar o dispositivo e dar-nos o direito de poder fazer consultas do Google Assistant para Conta do Google. Execute o seguinte comando no Raspberry Pi para instalar a ferramenta de autorização do Python.


python3  -m pip install --upgrade google-auth-oauthlib[tool]

5.2. Com a biblioteca de autenticação do Google agora instalada, precisamos executá-la. Para fazer isso, executaremos o seguinte comando no Raspberry Pi. Este comando irá gerar uma URL que irá precisar acessar no seu navegador da Web, portanto esteja preparado. 


google-oauthlib-tool --client-secrets ~/googleassistant/credentials.json \
--scope https://www.googleapis.com/auth/assistant-sdk-prototype \
--scope https://www.googleapis.com/auth/gcm \
--save --headless

5.3. Agora receberá o texto " Visite este URL para autorizar este aplicativo: " seguido por um URL muito longo. Certifique-se de copiar esse URL inteiramente no seu navegador da Web para abri-lo.

5.4. Nesta tela, faça login na conta do Google, se tiver várias contas, selecione a que configurou a chave de API. Posteriormente, deverá receber uma tela com o texto “ Copie este código, mude para o seu aplicativo e cole-o lá ” seguido por um longo código de autenticação. Copie o código de autenticação e cole-o novamente na sessão do terminal e pressione enter. Se a autenticação foi aceita, deve ver a seguinte linha aparecer na linha de comando: “ credenciais salvas: /home/pi/.config/google-oauthlib-tool/credentials.json ” .

5.5. Agora, com as credenciais de autenticação salvas, o Google ainda exige que concordemos com algumas coisas. Com esse comando, substitua "projectid" pelo ID do projeto. Se você não souber qual é o ID do projeto, acesse o Console de ações no Google, clique no projeto que criou e clique na roda dentada no canto superior esquerdo e depois em " Configurações do projeto ". Além disso, certifique-se de substituir "deviceid" pelo ID do dispositivo que obteve anteriormente no tutorial. Se perdeu o ID do dispositivo, pode encontrá-lo novamente acessando o site de Ações do console, clicando no projeto que criou e depois em " Registro do dispositivo " Para conversar com o assistente, pressione “ ENTER ” no terminal e fale uma ação como “ Qual é o tempo ”. Se tudo estiver funcionando como deveria, deverá ouvir uma resposta verbal do assistente. 


googlesamples-assistant-pushtotalk --project-id "projectid" --device-model-id "deviceid"

Se não receber nenhuma resposta do Assistente do Google, deve abrir uma nova guia que contenha os resultados, volte para a secção " Configuração do áudio para o Google Assistant " e a secção " Teste do áudio para o Assistente do Google " secção. Certifique-se de manter seu " ID do projeto " e " ID do dispositivo " à mão, pois ainda precisamos deles para utilizar o amostras.

6. Usando o Assistente do Google no Raspberry Pi



6.1. Agora que finalmente autorizamos totalmente o Raspberry Pi. Para fazer isso, execute o seguinte comando no seu Raspberry Pi.


source  env/bin/activate

6.2. Saberá se funcionou corretamente ao ver (env) aparecer na frente de cada linha. Com (env) aparecendo, pode fazer chamadas novamente para as amostras do Assistente do Google. Para iniciar a amostra push to talk, precisará executar o seguinte comando, desta vez não precisamos escrever no ID do projeto ou no ID do dispositivo, pois eles foram armazenados em cache quando utilizamos a ferramenta Push to Talk pela primeira vez.


googlesamples-assistant-pushtotalk

6.3. Além do exemplo push to talk, também há o código de exemplo hotword. Este código de exemplo funcionará ouvindo determinadas frases para ativá-lo. Por padrão, isso escutará " Ok Google " seguido do comando que deseja emitir. Iniciar a amostra de hotword é muito parecido com o push to talk, mas está a utilizar o utilitário googlesamples-assistant-hotword.


googlesamples-assistant-hotword --device-model-id "deviceid"

6.4. Felizmente, já deve ter configurado com sucesso o seu Raspberry Pi com o software Google Assistant. Se tiver problemas de áudio, recomendamos que volte para a secção " Configuração do seu áudio para o Google Assistant " e a secção " Teste do seu áudio para o Google Assistant ".
